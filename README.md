# RDL PERFORMANCE

## 📌 Sobre o Projeto

O projeto **RDL PERFORMANCE** consiste em um site institucional desenvolvido para um personal trainer que precisava fortalecer sua presença digital, apresentar seus planos de consultoria esportiva e captar novos clientes através da web.

link de acesso < https://lcsmaster13.github.io/projeto-rdl/index.html >

A plataforma foi criada com foco em experiência do usuário, design moderno e conversão de leads, permitindo que visitantes conheçam os serviços oferecidos e realizem inscrições diretamente pelo formulário integrado ao Google Sheets.

---

# 🎯 Objetivo do Projeto

O principal objetivo foi desenvolver uma página web moderna e responsiva capaz de:

- Apresentar os planos de treinamento;
- Divulgar os serviços da RDL PERFORMANCE;
- Captar potenciais clientes;
- Receber inscrições através de um formulário interativo;
- Armazenar automaticamente os dados dos usuários em uma planilha online.

# 🛠️ Ferramentas Utilizadas

## Site Stitch
<img width="1250" height="693" alt="image" src="https://github.com/user-attachments/assets/68103afa-f264-4cc2-b3d1-7677063bc2e2" />

Foi utilizada a plataforma **Site Stitch** para geração inicial das páginas HTML, CSS e JavaScript.

### Justificativa da escolha

A ferramenta foi escolhida pela praticidade e velocidade no desenvolvimento visual da interface, permitindo criar layouts modernos de forma rápida e eficiente.

Além disso, o código gerado possui boa compatibilidade com editores como o VS Code, facilitando personalizações manuais e integração com outras tecnologias.

### Limitações da ferramenta

Apesar da agilidade proporcionada pelo Site Stitch, algumas limitações foram identificadas:

- Necessidade de ajustes manuais no código;
- Estrutura HTML com elementos redundantes;
- Limitações em integrações mais avançadas;
- Necessidade de customização manual para funcionalidades específicas.

# 🌐 Aplicação dos Padrões Web

Durante o desenvolvimento foram aplicados os principais padrões da web:

## HTML

O HTML foi utilizado para estruturar semanticamente os elementos da página, organizando seções como:

- Cabeçalho;
- Hero section;
- Planos;
- Formulário de inscrição;
- Rodapé.

## CSS

O CSS foi responsável pela estilização visual da aplicação, incluindo:

- Paleta de cores;
- Responsividade;
- Tipografia;
- Espaçamentos;
- Efeitos visuais;
- Layout moderno utilizando Tailwind CSS.

## JavaScript

O JavaScript foi utilizado para:

- Manipulação do formulário;
- Validação de campos;
- Máscara de telefone;
- Interações visuais;
- Integração com a API do Google Sheets via Google Apps Script.

# ⚙️ Customizações Realizadas Manualmente

- Remoção de elementos HTML desnecessários;
- Reorganização da estrutura visual;
- Alteração da identidade visual e logo do projeto;
- Reposicionamento do formulário principal;
- Ajustes finos de CSS;
- Implementação manual da integração do formulário com Google Sheets utilizando JavaScript e Google Apps Script.

# 🔗 Integração do Formulário

O formulário de inscrição foi integrado ao Google Sheets utilizando:

- JavaScript (Fetch API);
- Google Apps Script;
- API Web App do Google.

# 📚 Aprendizados Obtidos

O desenvolvimento do projeto proporcionou diversos aprendizados sobre padrões web e plataformas low-code/no-code.

Os principais aprendizados foram:

- Entendimento prático da integração entre frontend e APIs;
- Estruturação de páginas utilizando HTML semântico;
- Manipulação de elementos com JavaScript;
- Customização avançada de layouts gerados automaticamente;
- Uso de ferramentas low-code para acelerar desenvolvimento;
- Integração entre GitHub Pages e serviços externos.

Além disso, ficou evidente como plataformas low-code aumentam significativamente a produtividade no desenvolvimento web, permitindo gerar estruturas complexas em poucos segundos, restando ao desenvolvedor apenas ajustes específicos e personalizações avançadas.
